# x402 Bridge

Package: `@drakkar.software/doubloon-bridge-x402`

Handles the [HTTP 402 Payment Required](https://x402.org) protocol. Unlike the other bridges, x402 payments are verified synchronously against a facilitator URL rather than via a webhook. The wallet address is extracted directly from the payment signature — no `WalletResolver` is needed.

---

## Installation

```bash
pnpm add @drakkar.software/doubloon-bridge-x402
```

---

## Configuration

```ts
import { X402Bridge } from '@drakkar.software/doubloon-bridge-x402';

const x402 = new X402Bridge({
  facilitatorUrl: 'https://facilitator.example.com',
  productResolver,
});
```

### `X402BridgeConfig`

| Field | Type | Required | Description |
|---|---|---|---|
| `facilitatorUrl` | `string` | ✓ | Base URL of the x402 payment facilitator. Used to construct payment-required response bodies. |
| `productResolver` | `StoreProductResolver` | ✓ | Maps x402 product identifiers to on-chain product IDs. See [Product Resolver](#product-resolver). |
| `logger` | `Logger` | | Optional structured logger. |

> **No `walletResolver`** — the paying wallet is extracted directly from the payment signature.

---

## Environment

x402 payments are always `'production'`. There is no sandbox/testnet concept in the current implementation.

---

## Product Resolver

```ts
interface StoreProductResolver {
  resolveProductId(store: string, storeSku: string): Promise<string | null>;
  resolveStoreSku(store: string, productId: string): Promise<string[]>;
}
```

`store` is `'x402'`. `storeSku` is the product identifier included in the x402 payment receipt. Return the 64-char hex on-chain product ID, or `null` (throws `PRODUCT_NOT_MAPPED`).

## Usage

x402 does not use the standard webhook flow. Instead:

1. Your server receives a request without a payment header → respond with `402` and a payment-required body.
2. The client pays and re-sends with the payment header.
3. Your server verifies the receipt and mints the entitlement.

```ts
// 1. Build the 402 response
const paymentRequired = x402.createPaymentRequired({
  productId:       'my-product',
  priceUsd:        4.99,
  durationSeconds: 30 * 24 * 3600, // 30 days
  description:     'Pro access',
});
res.status(402).json(paymentRequired);

// 2. On the follow-up request, verify and mint
const receipt: X402PaymentReceipt = {
  paymentId:       req.headers['x-payment-id'],
  wallet:          req.headers['x-payment-wallet'],
  productId:       'my-product',
  amountUsd:       4.99,
  durationSeconds: 30 * 24 * 3600,
  timestamp:       Date.now(),
};

const { notification, instruction } = await x402.verifyAndMint(receipt);
// Pass `instruction` to your chain writer
```

### `X402PaymentReceipt`

| Field | Type | Description |
|---|---|---|
| `paymentId` | `string` | Unique payment identifier from the facilitator. Used as deduplication key. |
| `wallet` | `string` | Paying wallet address (Solana base58 or EVM `0x...`). |
| `productId` | `string` | Product identifier included in the payment request. |
| `amountUsd` | `number` | Amount paid in USD. |
| `durationSeconds` | `number` | Duration of access in seconds. `0` = lifetime. |
| `timestamp` | `number` | Payment timestamp (epoch ms). |
| `raw` | `unknown` | Optional raw facilitator response for auditing. |

---

## `createPaymentRequired` Response Shape

```json
{
  "accepts":        ["x402"],
  "facilitatorUrl": "https://facilitator.example.com",
  "productId":      "my-product",
  "price":          { "amount": 4.99, "currency": "USD" },
  "durationSeconds": 2592000,
  "description":    "Pro access"
}
```

---

## Error Codes

| Code | Cause |
|---|---|
| `WALLET_NOT_LINKED` | Wallet address format invalid. |
| `PRODUCT_NOT_MAPPED` | `productResolver.resolveProductId` returned `null`. |
